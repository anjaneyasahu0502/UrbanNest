┌─────────────────────────────────────────────────────────────────────────────┐
│                            DATA FLOW                                         │
└─────────────────────────────────────────────────────────────────────────────┘

Registration Request
    │
    ├─▶ Validate Input
    │       │
    │       ├─▶ Check Username (unique)
    │       ├─▶ Check Email (unique, format)
    │       ├─▶ Check Mobile (unique, format)
    │       └─▶ Check Password (length, match)
    │
    ├─▶ Create User Record
    │       │
    │       └─▶ {
    │             username: "john",
    │             email: "john@example.com",
    │             mobile: "+1234567890",
    │             password: "$2b$12$...", // hashed
    │             emailVerified: false,
    │             mobileVerified: false
    │           }
    │
    ├─▶ Generate OTPs
    │       │
    │       ├─▶ emailOTP: "123456" (6 digits)
    │       ├─▶ mobileOTP: "654321" (6 digits)
    │       └─▶ otpExpiry: Date.now() + 10min
    │
    ├─▶ Send OTPs
    │       │
    │       ├─▶ Email Service ──▶ User's Email Inbox
    │       └─▶ SMS Service ───▶ User's Mobile Phone
    │
    └─▶ Store Session
            │
            └─▶ req.session.pendingUserId = user._id

Verification Request
    │
    ├─▶ Retrieve User from Session
    │
    ├─▶ Validate OTPs
    │       │
    │       ├─▶ Check Email OTP matches
    │       ├─▶ Check Mobile OTP matches
    │       └─▶ Check OTP not expired
    │
    ├─▶ Update User Record
    │       │
    │       └─▶ {
    │             emailVerified: true,
    │             mobileVerified: true,
    │             emailOTP: undefined,
    │             mobileOTP: undefined,
    │             otpExpiry: undefined
    │           }
    │
    └─▶ Clear Session & Redirect to Login

┌─────────────────────────────────────────────────────────────────────────────┐
│                         SECURITY LAYERS                                      │
└─────────────────────────────────────────────────────────────────────────────┘

Layer 1: Input Validation
    ├─▶ Client-side (HTML5 validation)
    └─▶ Server-side (Express validation)

Layer 2: Password Security
    ├─▶ Minimum length requirement
    ├─▶ Bcrypt hashing (12 rounds)
    └─▶ Confirmation match

Layer 3: OTP Security
    ├─▶ Random 6-digit generation
    ├─▶ 10-minute expiry
    ├─▶ Single-use (cleared after verification)
    └─▶ Separate OTPs for email and mobile

Layer 4: Session Security
    ├─▶ HTTP-only cookies
    ├─▶ SameSite protection
    └─▶ Secure flag (production)

Layer 5: Database Security
    ├─▶ Unique constraints
    ├─▶ Indexed fields
    └─▶ Verification flags

┌─────────────────────────────────────────────────────────────────────────────┐
│                         ERROR HANDLING                                       │
└─────────────────────────────────────────────────────────────────────────────┘

Registration Errors:
    ├─▶ Username exists ──▶ "Username already exists"
    ├─▶ Email exists ────▶ "Email already registered"
    ├─▶ Mobile exists ───▶ "Mobile number already registered"
    ├─▶ Password short ──▶ "Password must be at least 6 characters"
    └─▶ Password mismatch ▶ "Passwords do not match"

Verification Errors:
    ├─▶ Wrong email OTP ─▶ "Invalid email OTP"
    ├─▶ Wrong mobile OTP ▶ "Invalid mobile OTP"
    ├─▶ OTP expired ─────▶ "OTP expired" + Resend option
    └─▶ No session ──────▶ "No pending registration found"

Login Errors:
    ├─▶ Not verified ────▶ "Please verify your email and mobile number first"
    ├─▶ Wrong username ──▶ "Incorrect username"
    └─▶ Wrong password ──▶ "Incorrect password"
